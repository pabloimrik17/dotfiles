## Context

This dotfiles repo uses chezmoi to manage user configuration. Config files live under `dot_config/` in the source tree and deploy to `~/.config/` on target machines. Existing managed configs include Ghostty (`dot_config/ghostty/`) and Starship (`dot_config/starship.toml`).

OpenCode stores user-level config at `~/.config/opencode/`. Two files are relevant:

- `opencode.jsonc` — main config (model, autoupdate, providers, MCPs, agents)
- `tui.json` — TUI-specific config (theme, keybinds, scroll, diff style)

These are separate files with separate schemas (`config.json` and `tui.json` respectively). OpenCode merges config in precedence order: global < project, so project-level `opencode.json` files override global settings.

## Goals / Non-Goals

**Goals:**

- Manage `opencode.jsonc` and `tui.json` as chezmoi dotfiles under `dot_config/opencode/`
- Set default model (`anthropic/claude-opus-4-6`) and theme (`catppuccin-macchiato`)

**Non-Goals:**

- Managing MCP servers at user level (these stay per-project)
- Managing OpenCode runtime state (`kv.json`, `model.json`, `auth.json`, databases)
- Templating these files with chezmoi variables (no dynamic content needed for now)
- Managing `~/.config/opencode/package.json` (auto-generated by OpenCode for plugin deps)

## Decisions

**Plain files, not chezmoi templates**: Both files contain static JSON with no machine-specific values. Use plain `dot_config/opencode/opencode.jsonc` and `dot_config/opencode/tui.json` rather than `.tmpl` variants.

**JSONC for main config**: Use `opencode.jsonc` (not `.json`) to allow comments, matching the convention shown in OpenCode's documentation. The `$schema` reference provides IDE validation.

**Minimal config**: Only declare settings where we want an explicit default. OpenCode provides sensible defaults for everything else. This avoids config drift when OpenCode adds new options.

## Risks / Trade-offs

**chezmoi overwrites runtime changes** → If OpenCode ever writes to `opencode.jsonc` or `tui.json` at runtime, `chezmoi apply` would revert those changes. Mitigation: OpenCode writes runtime state to `kv.json` in `~/.local/state/`, not to the config files. Config files are read-only from OpenCode's perspective.

**`package.json` coexistence** → `~/.config/opencode/` already contains an auto-generated `package.json`. chezmoi will only manage the two config files and ignore `package.json` since it's not in the source tree.
