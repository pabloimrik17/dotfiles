## Context

Worktrunk provides `wt step copy-ignored` to copy gitignored files between worktrees using reflink (copy-on-write). It can be filtered via `.worktreeinclude`. The user config already handles dependency install (`post-create.deps` in `dot_config/worktrunk/config.toml`).

## Goals / Non-Goals

**Goals:**

- Copy `.claude/settings.local.json` to new worktrees automatically
- Use worktrunk's built-in `copy-ignored` mechanism

**Non-Goals:**

- Copying `node_modules/` (handled by existing user-level `post-create.deps`)
- Copying `.idea/` (per-workspace, causes conflicts)
- Copying `.husky/_/` (regenerated by `npm ci`)

## Decisions

### Use `.worktreeinclude` whitelist instead of copying all gitignored files

**Rationale**: Default `copy-ignored` copies everything. We only need `.claude/` — copying `.idea/` or `.husky/_/` would be wasteful or harmful.
**Alternative**: No `.worktreeinclude` (copy all) — rejected because `.idea/` workspace files would conflict across worktrees.

### Use `post-create` (blocking) instead of `post-start` (background)

**Rationale**: `wt switch --create branch -x claude` needs `.claude/settings.local.json` before Claude starts. `post-start` runs in background and may not finish in time.
**Alternative**: `post-start` — rejected because `-x` commands run after `post-create` but potentially before `post-start` completes.

### Project config (`.config/wt.toml`) not user config

**Rationale**: This is repo-specific (which files to copy depends on the project). User config already handles universal concerns (deps install, LLM commits).

## Risks / Trade-offs

- [Risk] User hooks run before project hooks → `deps` installs before `copy` runs → No conflict since they touch different files (node_modules vs .claude)
- [Risk] `.claude/settings.local.json` may diverge between worktrees if user modifies permissions → Acceptable, `copy-ignored` skips existing files by default
