## Why

The `.zshrc` sources an external file (`~/.fzf.zsh`) that is not tracked in the repo. This is the only CLI tool initialization that depends on an unmanaged external file — starship, zoxide, and atuin all initialize inline. The external file is a 7-line shim generated by `fzf`'s install script, and its contents can be replaced with a single `source <(fzf --zsh)` call plus an arch-aware PATH addition. Inlining it eliminates a hidden dependency and makes the repo fully self-describing.

## What Changes

- Replace `[ -f ~/.fzf.zsh ] && source ~/.fzf.zsh` in `dot_zshrc.tmpl` with inline fzf initialization
- Add chezmoi template conditionals for fzf PATH (ARM vs Intel macOS), consistent with existing zsh plugin path handling
- Use `source <(fzf --zsh)` for keybindings and completions (modern fzf 0.48+ API, auto-updates with installed version)
- The external `~/.fzf.zsh` file is no longer needed after this change
